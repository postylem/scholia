# Scholia v0.3: Global Packaging & Agent Integration

**Goal:** Make scholia installable as a global CLI tool and usable with any AI coding agent (Claude Code, Codex, Cursor, opencode, etc.) in any project directory.

## Scope

1. Verify and document global installation via `pipx` / `uv tool`
2. Auto-port selection for conflict-free multi-instance use
3. `scholia init` command to write agent instructions into a project
4. Agent-agnostic instruction template shipped inside the package
5. Install/usage documentation (README)

**Not in scope:** PyPI publishing, global config file (`~/.config/scholia/`), per-project `.scholiarc`.

---

## 1. Global Installation

The existing `pyproject.toml` entry point (`scholia = scholia.cli:main`) and `Path(__file__).parent` resource resolution already support global install. Static files (CSS, JS, templates) ship inside the `scholia/` package directory.

**Install methods:**

```bash
# From local clone
pipx install .
uv tool install .

# From git (no clone needed)
pipx install git+https://github.com/<user>/scholia.git
uv tool install git+https://github.com/<user>/scholia.git
```

**External dependency:** Pandoc must be installed separately. The CLI already checks for it at startup and prints a helpful error.

**Verification:** After install, `scholia start <any-path>/doc.md` should serve correctly from any working directory.

## 2. Auto-Port Selection

Currently `--port` defaults to 8088. When running multiple instances, users must manually pick ports.

**Change:** Add `--port 0` support. When port is 0, the OS assigns a free port. The server prints the actual URL after binding.

This is a one-line change in `server.py` — after `site.start()`, read the bound port from the socket and print it. The default remains 8088 for single-instance simplicity.

## 3. `scholia init`

Writes an AI agent instruction file into the current project.

```bash
# Default: writes .claude/skills/scholia.md
scholia init

# Custom path for other agents
scholia init .cursor/rules/scholia.md
scholia init AGENTS.md
```

**Behavior:**
- Creates parent directories if needed
- Skips if file already exists (prints message), unless `--force`
- Prints what file was written and a one-line explanation of what it does
- `--global` variant writes to `~/.claude/skills/scholia.md` (or custom path)

**The instruction content** is a single markdown template bundled in the scholia package (`scholia/data/agent-instructions.md`). It is agent-agnostic: says "AI assistant" not "Claude", describes the CLI commands and review workflow generically. The template is the same regardless of where it's placed.

## 4. Agent Instruction Template

The template teaches any AI coding agent how to use scholia. Content:

- **Context:** What scholia is, what `.scholia.jsonl` files are
- **CLI commands:** `scholia list`, `scholia reply`, `scholia comment`, `scholia resolve`, `scholia start`
- **Review workflow:** Read the doc, list open comments, respond to each, edit the doc when requested
- **Rules:** Don't edit `.scholia.jsonl` directly, keep replies concise, don't respond to resolved threads

This is essentially the current `CLAUDE.md` content, rewritten to be agent-neutral.
